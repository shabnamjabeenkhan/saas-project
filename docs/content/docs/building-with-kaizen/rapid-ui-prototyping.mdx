## The v0 + Cursor Workflow (Recommended)

### Overview
The most efficient way to build UIs in Kaizen is combining [v0](https://v0.dev) (Vercel's AI-powered UI generator) with Cursor's AI-assisted development environment.

### Step 1: Design with v0
1. **Describe your component** in natural language to v0
2. **Get instant UI mockups** with working code
3. **Iterate rapidly** by describing changes
4. **Copy the generated code** to your project

### Step 2: Integrate with Cursor
1. **Paste the code** into your component files
2. **Use Cursor's AI** to adapt the code to your existing design system
3. **Fix any integration issues** with AI assistance
4. **Test and refine** the component

### Example Workflow
```bash
# Generate a pricing page component with v0
"Create a modern pricing page with 3 tiers, featuring monthly/yearly toggle, popular plan highlighting, and smooth animations"
```

Copy the generated code and paste it into `app/components/pricing-page.tsx`, then use Cursor to:
- Adapt it to use your existing Tailwind config
- Integrate with your Convex data layer
- Add proper TypeScript types
- Ensure responsive design

## UI Component Library

Kaizen comes with a comprehensive UI component library built on shadcn/ui and Tailwind CSS.

### Available Components
- **Forms**: Input, Select, Checkbox, Radio, Textarea
- **Feedback**: Alert, Toast, Loading states
- **Layout**: Card, Tabs, Accordion, Sheet
- **Navigation**: Breadcrumb, Pagination, Sidebar
- **Data Display**: Table, Badge, Avatar, Tooltip

### Usage Example
```tsx
import { Card, CardHeader, CardTitle, CardContent } from "@/components/ui/card";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";

export function UserProfile({ user }) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>{user.name}</CardTitle>
      </CardHeader>
      <CardContent>
        <Input placeholder="Email" value={user.email} />
        <Button>Update Profile</Button>
      </CardContent>
    </Card>
  );
}
```

## Design System Principles

### Color Palette
- **Primary**: Blue (`#3b82f6`) for CTAs and important actions
- **Secondary**: Gray (`#6b7280`) for supporting text and borders
- **Success**: Green (`#10b981`) for positive actions
- **Warning**: Yellow (`#f59e0b`) for caution states
- **Error**: Red (`#ef4444`) for errors and destructive actions

### Typography Scale
- **Display**: 3rem (48px) - Page titles
- **Heading 1**: 2rem (32px) - Section headers
- **Heading 2**: 1.5rem (24px) - Component headers
- **Body Large**: 1.125rem (18px) - Important body text
- **Body**: 1rem (16px) - Regular body text
- **Body Small**: 0.875rem (14px) - Supporting text

### Spacing Scale
Use consistent spacing units: 0.25rem, 0.5rem, 0.75rem, 1rem, 1.5rem, 2rem, 3rem, 4rem, 6rem, 8rem

### Component Patterns

#### Loading States
```tsx
import { Skeleton } from "@/components/ui/skeleton";

function LoadingCard() {
  return (
    <Card>
      <CardHeader>
        <Skeleton className="h-4 w-[250px]" />
        <Skeleton className="h-4 w-[200px]" />
      </CardHeader>
      <CardContent>
        <Skeleton className="h-4 w-full" />
        <Skeleton className="h-4 w-3/4" />
      </CardContent>
    </Card>
  );
}
```

#### Error Boundaries
```tsx
import { Alert, AlertDescription } from "@/components/ui/alert";

function ErrorFallback({ error }) {
  return (
    <Alert variant="destructive">
      <AlertDescription>
        Something went wrong: {error.message}
      </AlertDescription>
    </Alert>
  );
}
```

  width={800}

### Code Splitting
```tsx
import { lazy, Suspense } from "react";
import { Skeleton } from "@/components/ui/skeleton";

const HeavyComponent = lazy(() => import("./HeavyComponent"));

function MyPage() {
  return (
    <Suspense fallback={<Skeleton />}>
      <HeavyComponent />
    </Suspense>
  );
}
```

## Testing UI Components

### Unit Tests with Vitest
```typescript
// components/Button.test.tsx
import { render, screen } from "@testing-library/react";
import { Button } from "./Button";

describe("Button", () => {
  it("renders children correctly", () => {
    render(<Button>Click me</Button>);
    expect(screen.getByText("Click me")).toBeInTheDocument();
  });
});
```

### Visual Regression Testing
Use tools like Chromatic or Playwright to catch visual regressions automatically.

## Deployment Checklist

Before deploying your UI changes:

- [ ] Run `bun run typecheck` to ensure type safety
- [ ] Test on multiple screen sizes (mobile, tablet, desktop)
- [ ] Check for accessibility issues with screen readers
- [ ] Verify loading states work correctly
- [ ] Test error boundaries with invalid data
- [ ] Run performance audits with Lighthouse
- [ ] Check for console errors in production build
